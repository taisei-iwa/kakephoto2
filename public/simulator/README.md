# かけフォト 掛軸オーダーシミュレーター（kakephoto-order）

かけフォト（KAKEPHOTO）のお客様向け 掛軸オーダーシミュレーターです。お手元の写真を本紙に当てはめ、掛軸の各部の裂地をプレビュー上で選びながら、サイズに応じたお見積もりの目安をその場で確かめられます。

- レーン: かけフォト（KAKEPHOTO）。岩﨑精正堂（表具師業）との関係は画面・コード・コメントに一切出しません。
- 派生元: 個人用の `kakejiku-toriawase/`（掛軸プレビュー・本紙への写真取り込み・部位クリックの裂地ピッカー）を土台にコピーして顧客向けへ作り替えたものです。派生元は変更していません。
- 例示の写真・裂地名はすべてダミーです。

## 起動方法

依存ゼロ。Node 標準モジュールのみで動く静的サーバーです。

```
cd /Users/t/Claude_Code_Minimum_Harness/kakephoto-order
node server.js
```

または:

```
npm start
```

- URL: http://localhost:5175
- ポート: 既定 5175（`PORT` 環境変数で変更可）。
- 注意: 既存の個人用 `kakejiku-toriawase`（既定 5174）とは別ポートです。5174 を止めずに動かせます。

API キー・環境変数は不要です。外部送信もありません（注文導線の遷移先 URL は後続スプリントで設定）。

## 実装済み（スプリント2 時点）

### スプリント1 実装済み

- 写真（本紙）の取り込みと本紙への当てはめ（F1）。写真は本紙領域内にクリップされ、外へ漏れません。別の写真への差し替え・取り外し・非画像ファイルの警告に対応。
- サイズ選択（F2）:
  - A4 定型（縦・横）
  - A3 定型（縦・横）
  - A3 までの自由サイズ（mm 数値入力。A3 = 420 × 297 mm が上限。超過・0 以下は警告して確定しない）
- サイズ価格のリアルタイム表示（F3）:
  - A4 定型 = ¥46,000 / A3 定型 = ¥77,000
  - 自由サイズは「収まる最小の定型枠に切り上げ」: 〜B5 = ¥33,000 / 〜A4 = ¥46,000 / 〜A3 = ¥77,000
  - 「お見積もりの目安」「税込・送料別」と明記
- 中央プレビューと部位クリックの裂地ピッカー（F4・F5 の表示部分）。表装形式は全形式（三段表装・行／茶掛／丸表装／明朝仕立て）を表示。部位オプション（一文字なし／風帯なし）は形式を変えても状態を保持します。
- 箱4（裂地の取り込みフォーム）・箱5（裂地候補一覧グリッド）は顧客画面に表示しません。
- スマホ・タブレット幅で主要要素が横スクロールなしで縦積み表示（F9 部分）。

### 裏方: 裂地の追加/削除ツール（2026-06-13、catalog.js 自動更新方式に刷新）

- コード編集なしで裂地を追加できる裏方専用パネル。**URL の末尾に `#admin` を付けたときだけ表示**（例: `http://localhost:5175/#admin`）。お客様の通常 URL には出ない。`#admin` 時はスマホでも全体スクロール可（`body.admin-mode`）。
- 取り込み時に**トリミング**できる（本紙の位置調整と同じ仕組みを汎用化して流用。`openTrimDialogWith()`、本紙比率固定は非表示の自由トリミング）。柄が継ぎ目なく繰り返す範囲を切り抜く想定。
- パネルから「画像・名前・用途（天地／中廻し・柱／一文字）・等級・柄の大きさ(px)」を指定して「追加する」。
- 追加した裂地は一覧から**編集**もできる（「編集」ボタン → フォームに展開 → 「更新する」。名前・用途・等級・柄の大きさを変更可。「編集をやめる」で追加モードに戻る）。
- 画像は2通りで変更できる: ①新しいファイルを選び直す（=差し替え＋トリミング）、②**「切り抜きを調整」ボタンで現在の画像を再トリミング**（保存済みスワッチは同一オリジン配信なので canvas で切り出し可）。再トリミングは「今より狭く切り直す」調整向け。元より広い範囲に戻すには元画像を選び直す（元の未トリミング画像は保持しないため）。
- **サーバー経由で `js/catalog.js` を自動更新**（`server.js` の API）:
  - `POST /api/fabric/add` … 画像を `assets/fabrics/custom_<ts>.png` に保存し、`FABRIC_CATALOG` に1行追記（id は `cust_<ts>`）。
  - `POST /api/fabric/update` … 該当行を作り直す。`dataUrl` があれば画像差し替え（新規保存＋旧削除）、無ければ画像据え置き（`cust_` のみ）。
  - `POST /api/fabric/delete` … 該当行を削除し画像も削除（`cust_` のみ）。
  - 追加/編集/削除後は実行中ページにも即時反映。一覧は `cust_` の裂地のみ表示。割り当て済みを削除した場合は自動で解除。
- **重要**: API はローカルで `node server.js` を動かしているときだけ機能（静的ホストは書き込み不可）。運用は「**ローカルで追加 → 再デプロイ**」。これで全お客様に反映される。
- 画像はトリミング後に最大 400px へ縮小して保存。catalog.js への追記は JSON.stringify で安全にエスケープ。

### UI 改善（2026-06-13）

- 「LINE で相談する」ボタンを LINE ブランドカラー（`#06C755` / ホバー `#05B249`・白文字）にし、白いトークふきだしの SVG アイコンを追加（`.cta` を flex 中央寄せ化）。「フォームで相談する」は従来の臙脂アウトラインのまま。
- プレビューの裂地にカーソルを重ねると、同じ裂地として一緒に変わる部位をまとめて枠で囲む（ホバー強調）:
  - グループは割り当てと同一基準（`buildAssignmentUiMap()`）。天↔地、中廻し上下＋柱左右、一文字上下＋風帯（形式の `linkGroups` を反映）。本紙（写真）は対象外。
  - 各部位に `data-hl-group` を持たせ、`#preview` への委譲ハンドラで同グループに `.hl` を付与（CSS は `.part.hl`）。ポインタ操作向けの補助表示。
- プレビューの中央寄せを厳密化（PC・スマホ問わず、サイズ未選択の初期状態を含め水平中央）:
  - 高さラベルと右側の釣り合いスペーサーを常に同じ固定幅にし、ラベルの文字有無に依らず中央を保つ。
  - 左右パネル幅を 300px で統一し、中央プレビュー列をページ中央に一致させる。
- スマホ・タブレット幅（≤960px）を 1 画面に収める再設計（縦スクロール不要）:
  - `100dvh` のグリッドで「左:操作 / 右:プレビュー(大) / 下:お見積りバー」を配置。掛軸は縦長なので、右側で縦いっぱいに使い大きく見せる（左 150px 固定・右は残り幅）。
  - プレビューは `js/app.js` の `render()` が `.preview-frame` を実測し、掛軸全体が枠に収まるよう縮小して描き直す（2 段階描画。拡大はしない）。列内では中央寄せ。
  - 操作（写真・大きさ・仕立て・箱）は左列に縦積みで整頓。サイズの A4/A3/自由は縦積みボタン、チェックボックスも縦並び。列が溢れる小型端末では左列だけ内部スクロール（ページ全体はスクロールしない）。
  - 「お見積りの目安」ボックスは下段に全幅でそのまま（合計＋税込・送料別＋CTA）。
  - 効果: 縦 844px 端末でプレビュー高さ約 470px、667px 端末でも約 450px（旧縦積みは ~140px）。

### UI 改善（2026-06-20）

- スマホで軸先（端の飾り）が細い帯に切れていたのを修正:
  - 軸先（左右 ~20px）・軸棒（下 ~10px）は `#preview` の外へ食み出すが、スマホの `.preview-frame { overflow: hidden }` がその食み出しを切り、軸先が 14×14 の正方形から細い帯に見えていた。
  - `.preview-frame` に内側余白（`padding: 10px 22px 12px`）を確保し、食み出しぶんを枠内に収めてから中央寄せする方式に変更。軸先が PC と同じ正方形で表示される。1 画面に収まる挙動は維持。
- 「裂地をすべてリセット」ボタンを追加:
  - 全部位の裂地割り当てを一括で無地に戻す（`state.assignments = {}` → `render()` + `updatePrice()`）。
  - 裂地が 1 つも割り当てられていないときは `disabled`（`render()` で全 `.fabric-reset-btn` を判定）。
  - 配置は端末で出し分け。PC は仕立てブロック内（`.fabric-reset-pc`）、スマホはプレビューの下（`.preview-area` 内の `.fabric-reset-sp`）に置く。左の操作列とプレビュー列は別グリッドエリアのため CSS の並べ替えでは移せず、2 ボタンを `@media` で表示切替する方式。動作（クリック・有効/無効）はクラス `.fabric-reset-btn` で両方にまとめて適用。
  - スマホ用はプレビュー列の縦フレックス内に入るため `flex: none`、折り返さないよう `white-space: nowrap`。
- スマホで自由サイズを選ぶと操作列の下部（軸先の色・箱）が触れなくなる不具合を修正:
  - 原因: `.layout` は `align-items: flex-start`（PC 用）のため、スマホグリッドの `.panel-controls` がトラック高にストレッチされず内容高のまま 1fr 行を突き抜け、`overflow-y: auto` が効かずお見積りボックスと重なっていた。
  - 対応: `.panel-controls`（`@media (max-width: 960px)`）に `align-self: stretch` と `min-height: 0` を付与。操作列がトラック高に収まり、溢れたぶんはこの列だけ内部スクロールする。お見積りボックスは別行（`grid-template-rows: 1fr auto` の `auto` 行）なので常に下部固定のまま。
- プレビューの文言を変更（PC/スマホで動詞を出し分け）: 「掛軸を**クリック/タップ**して裂地をお選びください」（動詞は太字、PC=クリック・スマホ=タップ。`index.html` の `.act-click`/`.act-tap` を `@media` で切替）。

## データ構造（編集可能データの置き場）

価格・裂地メタ・等級・箱の金額は `js/catalog.js` に集約しています（顧客画面のハードコードではありません）。

- `SIZE_TIERS` / `FIXED_SIZES`: サイズ枠の実寸と価格
- `FABRIC_GRADES`: 等級ラベルと加算額（標準 / 上 / 特上）
- `BOX_OPTIONS`: 箱の種類と加算額（桐箱 +¥11,000 確定 / 2026-06-13）
- `FABRIC_CATALOG`: 顧客に見せる裂地候補（画像・名前・用途・等級）

価格計算ロジックは `js/pricing.js`（合計 = サイズ価格 + 裂地加算 + 箱 の内訳合算構造）。
